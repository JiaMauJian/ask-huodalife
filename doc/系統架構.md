# 小豁 1.1 — 系統架構文件

> 豁達人生財經室知識助理，基於 RAG（Retrieval-Augmented Generation）架構

## 系統總覽

```
使用者提問
    │
    ▼
┌─────────────────────────────────────────────────┐
│  前端（index.html）                              │
│  - 輸入問題、顯示串流回答                         │
│  - 歷史記錄（localStorage）                       │
│  - Claude 分析模板（一鍵複製 + 開啟 Claude）       │
└─────────────────┬───────────────────────────────┘
                  │ POST /api/ask
                  ▼
┌─────────────────────────────────────────────────┐
│  Vercel Serverless（api/ask.py）                  │
│  - 接收問題                                       │
│  - 呼叫 blog_qa.py 的檢索 + 生成流程              │
│  - SSE 串流回傳回答                               │
│  - 寫入 Supabase log                             │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│  blog_qa.py（核心問答引擎）                        │
│                                                   │
│  1. 混合搜尋（hybrid_search）                     │
│     ├─ 語意搜尋：Voyage API → cosine similarity   │
│     └─ 關鍵字搜尋：bigram 比對（fallback）         │
│     → 語意 70% + 關鍵字 30% 混合計分              │
│                                                   │
│  2. Haiku 選文（select_top_articles）              │
│     → 從候選 20 篇中選出最相關 3 篇               │
│                                                   │
│  3. Sonnet 回答（stream_answer / generate_answer） │
│     → 根據完整文章內容 + soul.md 生成回答          │
└─────────────────────────────────────────────────┘
```

## 資料流程

### A. 建置階段（離線執行）

```
blog_scraper.py
    │  爬取 pixnet 部落格（月報、年報、好書分享、交易紀錄）
    │  儲存為 articles.json
    ▼
blog_indexer.py
    │  1. 讀取 articles.json + special/*.json
    │  2. Haiku 產生每篇 150 字摘要
    │  3. Voyage API 產生每篇 embedding 向量（1024 維）
    │  4. 儲存為 blog_index.json
    ▼
blog_index.json
    每筆資料結構：
    {
      "id": "文章ID",
      "title": "標題",
      "date": "日期",
      "url": "連結",
      "category": "分類",
      "summary": "150字摘要",
      "embedding": [0.012, -0.005, ...]  ← 1024 個浮點數
    }
```

### B. 查詢階段（線上即時）

```
使用者問題
    │
    ▼
┌─ hybrid_search ─────────────────────────────┐
│                                              │
│  ┌─ 語意搜尋（70% 權重）──────────────────┐  │
│  │  問題 → Voyage API → query embedding   │  │
│  │  和 blog_index.json 中每篇的 embedding  │  │
│  │  算 cosine similarity，取最相似的       │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌─ 關鍵字搜尋（30% 權重）────────────────┐  │
│  │  問題拆成 bigram（每 2 字一組）         │  │
│  │  比對 title + summary 中的命中次數      │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  兩者分數正規化後加權合併 → 取 top 20 候選  │
└──────────────────────┬───────────────────────┘
                       ▼
              Haiku 選出 top 3 篇
                       │
                       ▼
              從 articles.json / special/
              撈出完整文章內容
                       │
                       ▼
              Sonnet 根據 soul.md + 文章內容
              串流生成回答
                       │
                       ▼
              SSE 串流回傳前端
```

## 建置 / 更新流程

### 新增文章（定期執行）

```bash
# 1. 爬取新文章（自動跳過已爬的）
python blog_scraper.py

# 爬蟲結束後會自動呼叫 blog_indexer.py
# 產生摘要 + embedding，寫入 blog_index.json
```

### 新增特殊知識

```bash
# 1. 把 JSON 檔放進 special/ 資料夾
# 2. 重跑 indexer
python blog_indexer.py

# 3. 部署
git add . && git commit -m "update index" && git push
```

### 首次建置

```bash
# 1. 設定環境變數（.env）
# 2. 爬取所有文章
python blog_scraper.py
# 3. 確認 blog_index.json 中所有文章都有 embedding
python blog_indexer.py
# 4. 部署到 Vercel
vercel --prod
```

## soul.md 的角色

`soul.md` 是小豁的人格核心，每次 Sonnet 生成回答時都會被放進 prompt。它定義了：

- 個性（理性冷靜、簡潔有力）
- 投資哲學（順勢操作、汰弱留強、比較原則）
- 回答規則（只根據文章回答、不自行發揮）
- 引導機制（遇到圖表問題引導去看原文和工具）
- 三種成長股策略摘要（穩定型、長期強勢型、短期強勢型）
