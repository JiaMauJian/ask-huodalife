Claude API 使用指南
本文整理 Claude API 的提示詞技巧與費用優化策略，幫助開發者獲得更好的回應品質並降低使用成本。

提示詞的藝術（The Art of Prompting Claude）
無論是剛開始使用還是已在使用 Claude API，回應品質很大程度取決於提示詞的結構。提示詞工程是從 Claude API 獲取價值最重要的技能之一，以下三個技巧能有效提升結果：

1. 清楚、直接、詳細
•	Claude 在獲得具體指示時表現最佳
•	避免模糊語句，明確說明你期望的輸出格式與內容範圍

2. 提供背景脈絡（Context）
•	說明任務的目的與重要性，幫助 Claude 提供更有針對性的回應
•	包含相關的背景資訊、使用情境或目標受眾

3. 使用範例（Examples）
•	在提示詞中加入 2~3 個範例，能顯著提升準確性與一致性
•	範例可以是輸入輸出對照、格式示範或風格參考

費用優化策略（Cost Optimization Strategies）
在使用 Claude 建構 Agent 應用時，以下策略能有效控制 API 費用：

1. 選用適當的模型
•	簡單任務（關鍵字擴充、分類篩選）使用 Haiku，費用較低
•	複雜推理與正式回答使用 Sonnet，品質較高

2. 實作 Prompt Caching
•	對重複出現的固定內容（如 system prompt）加上 cache_control 標記
•	快取命中時費用僅為原價的 10%，5 分鐘內有效
•	使用量越密集，效果越顯著

3. 批次處理（Batch API）
•	對非即時性任務使用 Batch API，費用更低
•	適合大量文章摘要、索引建立等背景作業

4. 監控使用量（Monitor Usage）
•	追蹤每次請求的 token 消耗，找出優化機會
•	設定月消費上限與自動補點，避免服務中斷
•	高流量應用可聯繫 Anthropic 企業團隊取得自訂方案
